# 工作协议

## 会话开始

- 先读 `AGENTS.md`
- 再读 active work
- 再读相关 docs
- docs 不够时才查 KB

## 任务接入

将任务归类为：

- `frontend-only`
- `backend-only`
- `contract-first`
- `full-stack`
- `data-affecting`
- `ops-affecting`

## 修改前定位

- 先从 docs 定位
- docs 不够时查询 KB
- 最后再做大范围仓库搜索

## 会话收口

- 更新 active work
- 结构变化时刷新 KB
- 长期认知变化时更新 docs
